# Templates

This directory exists to provide directly usable ChaOS artifacts.

## Purpose

Templates help humans and agents apply ChaOS without starting from a blank page. They are fillable documents, not applications or automation systems.

## Fit Within ChaOS

Templates sit between the constitutional documents and project-specific work. They help capture context, preserve assumptions, and route projects into the correct ChaOS workflow.

Templates must remain plain, inspectable, and portable. They must not require a database, frontend, form tool, vendor service, or workflow engine.

## Available Templates

- `architecture-starter-package.md`: Captures the standard output of Architecture Generation for human review.
- `project-intake-packet.md`: Captures raw business context and routes the project to Context Sufficiency Assessment, Architecture Generation, or no generation yet.

## Examples Of Use

A founder can complete the Project Intake Packet before asking ChaOS to generate architecture.

A project team can review an Architecture Starter Package before deciding whether implementation planning is allowed.

A future project can store a completed packet as evidence for why Architecture Generation was ready, partial, or blocked.

A coding agent can inspect a completed packet before making recommendations.

## Future Considerations

Additional templates must be added only when repeated use proves that one reusable intake packet is not enough.
