# Glossary

This document defines recurring ChaOS terms. It exists to reduce hidden context and make the repository easier to inherit.

## Agent

A bounded system component that uses defined inputs and processing logic to produce structured outputs. An agent must have evaluation criteria and approval boundaries.

## Architecture

The durable structure of a system: its concepts, responsibilities, decisions, feedback loops, and constraints.

## Decision

A chosen interpretation, recommendation, classification, priority, or next step.

## Entity

The thing a system cares about and tracks over time.

## Feedback

Structured learning from outcomes, human judgment, failures, and evaluations.

## Outcome

The observed result after a decision or action.

## Signal

An observed fact, event, pattern, or change that may influence a decision.

## Workflow

A repeatable path for work:

Trigger -> Input -> Processing -> Decision -> Action -> Feedback

## Future Considerations

The glossary should expand only when a term becomes important across multiple documents or future projects.
