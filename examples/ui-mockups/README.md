# UI Mockups

This directory exists to hold visual product explorations that may help future projects discuss interface direction.

## Purpose

UI mockups make abstract architecture easier to inspect as a possible user experience. They are static examples, not product code.

## Fit Within ChaOS

Mockups belong in `examples/` because they demonstrate how ChaOS patterns might appear in a future product without changing the ChaOS constitution.

They must not define implementation requirements, frontend standards, product scope, or vendor choices.

## Examples Of Use

A founder can use a mockup to discuss product tone before implementation planning begins.

A designer can use a mockup to test whether an architecture feels understandable on screen.

A future agent can review a mockup for alignment with ChaOS laws before recommending product direction.

## Future Considerations

If UI mockups become frequent, ChaOS may add a naming convention or review checklist for them. That must happen only after repeated use shows the added structure would remove more complexity than it adds.
