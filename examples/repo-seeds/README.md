# Repo Seeds

This directory contains repo-ready examples that may become separate projects.

## Purpose

Repo seeds let ChaOS explore the shape of a future repository without mixing that future repository into the ChaOS constitution.

## Fit Within ChaOS

Repo seeds are examples. They do not create ChaOS architecture, implementation requirements, product scope, or governance decisions by themselves.

They must be moved into a separate repository before they become active implementation work.

## Examples Of Use

A repo seed can define a starting structure for a new tool-based agent library.

A future maintainer can inspect a repo seed before deciding whether to create a standalone GitHub repository.

An agent can use a repo seed as a draft contract, then refine it after human review.

## Future Considerations

If a repo seed becomes active, ChaOS must record whether the pattern belongs back in ChaOS or remains project-specific.
